# 分层抽样的两阶段词汇量测试方案

## 1. 概述

本方案基于 **Rasch 模型（单参数逻辑斯蒂模型）**，对已有 `stage_vocab.json`（11950 词，含 difficulty 评分）和 `VocabBank`（21738 词，含 wordfreq rank）进行概率建模，实现两阶段自适应词汇量测试。

### 数据结构

| 资产 | 规模 | 关键字段 |
|------|------|---------|
| `stage_vocab.json` | 11950 词 | difficulty [0,1], cluster_20 [0-19], cluster_100 [0-99] |
| VocabBank | 21738 词 | wordfreq rank (1=最常见) |

每个 cluster_20 约 597~598 词，每个 cluster_100 约 119~120 词。
每个 cluster_20 对应 5 个连续的 cluster_100（如 c20[0] → c100[0..4]）。

## 2. 传统方法 vs Rasch 方法

### 2.1 传统比例法（M/N 法）

```
class_estimate = class_size × (correct / asked)
total_vocab    = Σ class_estimate
```

**缺点：**
- 小样本时方差极大：3 题中 1 对 → 33% 置信度，但该类比 33% 高或低很多都有可能
- 无法处理"答对简单词 + 答错难词"的矛盾信息
- 需要人为设定 rank 阈值来切分 bucket，切法影响结果
- 置信区间依赖 bootstrap，计算量高

### 2.2 Rasch 模型法

Rasch 模型假设每个词 `j` 有一个难度参数 `d_j`，用户有一个能力参数 `θ`：

```
P(word_j known | θ) = σ(θ - d_j)

其中 σ(x) = 1 / (1 + exp(-x))   (逻辑斯蒂函数)
```

**核心思想：**
- 用户能力 θ 在 logit 尺度上，与词难度 d_j 在同一空间
- 当 θ > d_j 时，认识该词概率 > 0.5
- 当 θ < d_j 时，认识该词概率 < 0.5
- 当 θ ≈ d_j 时，概率 ≈ 0.5（最不确定，信息量最大）

**词汇量估计：**

```
vocab_size = Σ_j P(word_j known | θ)          （所有 21738 词）
```

这是**概率期望**，而非硬分类。平滑衰减替代了硬阈值。

### 2.3 数学对比

| 特性 | 传统比例法 | Rasch 模型法 |
|------|-----------|-------------|
| 参数空间 | 每类一个独立比例 | 单一 θ 参数 |
| 样本效率 | 需均匀覆盖所有类 | 可自适应聚焦 |
| 连续度 | 阶梯式（每类各有一个比例） | 平滑概率曲线 |
| 自洽性 | 答对简单+答错难词可能矛盾 | 自动找到最优 θ |
| 置信区间 | bootstrap 模拟 | Fisher 信息量解析计算 |
| CAT 支持 | 不支持 | 天然支持（信息量最大化） |

### 2.4 Rasch 损失函数与 MLE

给定回答 `{(word_j, d_j, y_j)}`，其中 `y_j ∈ {0, 1}`：

```
对数似然:  LL(θ) = Σ_j [y_j·log(σ(θ-d_j)) + (1-y_j)·log(1-σ(θ-d_j))]
```

最大化 LL(θ) 等价于最小化交叉熵损失：

```
Loss(θ) = -LL(θ)
```

使用 scipy.optimize.minimize 进行 MLE 拟合，同时计算 Fisher 信息量：

```
I(θ) = Σ_j σ(θ-d_j) · (1-σ(θ-d_j))

θ 的标准误:  SE(θ) = 1 / sqrt(I(θ))

95% 置信区间:  θ ± 1.96 × SE(θ)
```

词汇量置信区间通过对 θ 置信区间的上下界分别求和得到。

## 3. 采样策略

### 3.1 Phase 1：快速定位（60 题）

分为两个子阶段：

#### 3.1a 探索期（前 10 题）
- 从 cluster_20 的 {0, 5, 10, 15, 19} 各抽 2 词
- 覆盖从最易到最难的极值
- 目的：快速获取 θ 的粗略定位
- 每类内：选 difficulty 最低和最高的词（最大化类内覆盖）

#### 3.1b 聚焦期（后 50 题）
- 每回答一题后，用前 N 题回答重新拟合 θ_N
- 按 θ_N 筛选"信息量大"的 difficulty 类：
  - 计算每个 cluster_20 的信息量期望：
    ```
    I_c(θ) = Σ_{j∈class_c} σ(θ-d_j)·(1-σ(θ-d_j))
    ```
  - 选择信息量最大的 20 个词（密度最高区域）
- 同时保留 3 个"安全带"词（来自与当前 θ 相距最远的类），防止 θ 漂移

### 3.2 Phase 2：细化测试（可选，max 100 题）

对 Phase 1 中置信度低的 20 类（c=1 或 c=2，即 3 题中答对 1 或 2 题）：

- 每个低置信类对应 5 个 cluster_100
- 每 cluster_100 抽 2 词（尽量选信息量最大的词）
- 每个低置信类最多增加 10 题
- 全部 Phase 2 回答加入后，重新拟合 θ

### 3.3 信息量最大化采样（CAT）

关键思想：选**当前 θ 下最不确定**的词。

```
信息量(word_j | θ_N) = σ(θ_N - d_j) · (1 - σ(θ_N - d_j))
                     = 方差在 0.5 附近最大
```

这等价于选择 difficulty 最接近 θ_N 的词。这种自适应测试（Computerized Adaptive Testing）比固定分层抽样效率高 2-3 倍。

## 4. 置信度与置信区间

### 4.1 基于二项分布（传统方法）

对每个 cluster_20（k 题中 c 题答对）：

```
观察比例:  p_hat = c/k
标准差:    SE = sqrt(p_hat·(1-p_hat)/k)
置信度:    = 1 - (SE / 0.289)    （0.289 = 3 题最大的可能 SE，即 p=0.5 时）
```

当 k=3 时：
- c=0: p=0, SE=0, 置信度=1.0（高）
- c=1: p=0.333, SE=0.272, 置信度≈0.06（低）
- c=2: p=0.667, SE=0.272, 置信度≈0.06（低）
- c=3: p=1, SE=0, 置信度=1.0（高）

### 4.2 基于 Fisher 信息量（Rasch 方法）

```
词汇量:  V(θ) = Σ_j σ(θ - d_j)

敏感性:  V'(θ) = Σ_j σ(θ-d_j)·(1-σ(θ-d_j))

词汇量 SE:  SE_V = V'(θ) / sqrt(I(θ))

95% CI:  V(θ) ± 1.96 × SE_V
```

这比 bootstrap 更高效（解析解而非模拟）。

## 5. 词汇量校准

最终词汇量需要经过两阶段校准：

### 5.1 饱和压缩（tanh）

```
V_calibrated = 20000 × tanh(k × V_raw)

k = 0.0000691
```

使全答对（~21300）校准到 ~18049（母语水平）。

### 5.2 分段线性压缩

| 区间 | 斜率 | 效果 |
|------|------|------|
| [0, 3000] | 1.00 | 不变 |
| [3000, 8000] | 0.45 | 压缩 55%（四级/六级中间区） |
| [8000, 22000] | 1.28 | 扩展 28%（高级区） |

## 6. API 接口设计

### Phase 1: `GET /api/vocabulary/quiz-v2`

返回 60 题（自适应分层抽样）。

```json
{
  "questions": [...],
  "count": 60,
  "seed": 12345,
  "cluster_info": {"stages": [...], "classes": {"0": 3, "1": 3, ...}}
}
```

### Phase 2: `POST /api/vocabulary/quiz-v2-stage2`

传入 Phase 1 回答 + Phase 1 估计结果，返回细化题。

```json
{
  "responses": [{"word": "...", "known": true}],
  "phase1_result": {...}
}
```

返回：

```json
{
  "questions": [...],
  "count": 65,
  "low_confidence_classes": [3, 7, 12, 15],
  "stage2_details": {...}
}
```

### 词汇量估计: `POST /api/vocabulary/quiz-v2/estimate`

传入全部回答，返回 Rasch 估计结果。
