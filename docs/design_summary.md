# 设计总结

本次课程设计最初只是想做一个“根据认识多少单词估计词汇量”的工具，我一开始以为词频 rank 和答对比例就足够。真正实现后才发现，简单统计会把高频词正确率外推到低频词，尤其容易高估低词汇量用户。后来改用 Rasch 1PL IRT 模型，用用户能力 theta 和单词难度 d 描述认识概率，我也认识到词汇量估计本质上是带不确定性的测量问题。

项目中最难的是 difficulty 标注、分层采样和模拟评估。难度要同时考虑教育阶段和 wordfreq rank；采样要覆盖 20 个难度类，并把题目放到能力边界附近；评估必须用合成用户批量计算 MAE、R²、偏差和分桶误差。

过程中也遇到困难。Phase 2 设计更精细，但实验提升有限；词向量校正实验因训练目标和残差应用不一致而失败；seed 和题目顺序还会带来方差。最终我完整经历了需求、设计、实现、测试、部署和文档化流程，最大的收获是学会用实验闭环验证设计，而不是只凭直觉调参。
