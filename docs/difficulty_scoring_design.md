# 综合难易度分数设计

## 1. 设计目标

为 `stage_vocab.json` 中约 11,950 个词计算 [0, 1] 范围内的难易度分数，综合两个维度：

| 维度 | 含义 | 范围 | 方向 |
|------|------|------|------|
| **教育阶段** (stage_priority) | 教材/考纲体系的年级阶段 | 1 (小学3年级) ~ 11 (雅思) | 越高越难 |
| **词频排名** (wordfreq rank) | wordfreq 语料库的词频排名 | 1 ~ 30,000 | 排名越大 (越罕见) 越难 |

### 关键约束

1. **区分度优先**: 同阶段内 (例如 senior 的 4,536 个词), wordfreq 要能拉开至少 0.1~0.3 的差距
2. **单调性**: 更晚阶段 + 更高 rank 必须得到更高分数
3. **平衡性**: 两个维度都不能被淹没

## 3. 归一化方案

### 3.1 Stage Priority → norm_stage

**方案**: 线性 Min-Max 归一化

```
norm_stage = (priority - 1) / (max_priority - 1) = (priority - 1) / 10
```

| priority | norm_stage |
|:--------:|:----------:|
| 1 | 0.000 |
| 5 | 0.400 |
| 8 | 0.700 |
| 11 | 1.000 |

### 3.2 Wordfreq Rank → norm_rank

词频排名服从 Zipf 分布 (指数分布 → 大量词集中在低频端)。**线性归一化**会将 rank 1~1000 全部压缩到 ~0.03 内，失去区分度。

**推荐: Log 归一化**

```
norm_rank = log(rank + 1) / log(max_rank + 1) = log(rank + 1) / log(30001)
```

| rank | norm_rank (log) | norm_rank (linear) | norm_rank (sqrt) |
|:----:|:---------------:|:------------------:|:----------------:|
| 1 | 0.067 | 0.000 | 0.006 |
| 100 | 0.448 | 0.003 | 0.058 |
| 500 | 0.603 | 0.017 | 0.129 |
| 1,000 | 0.670 | 0.033 | 0.183 |
| 5,000 | 0.826 | 0.167 | 0.408 |
| 10,000 | 0.893 | 0.333 | 0.577 |
| 20,000 | 0.961 | 0.667 | 0.816 |
| 30,000 | 1.000 | 1.000 | 1.000 |

**Log 优势**: 对 rank 1~1000 这个词汇覆盖最密集的区域有较好的拉伸，同时不压缩高频端。

## 4. 权重设计

### 4.1 候选公式

#### A. 加权平均 (推荐)

```
difficulty = α × norm_stage + β × norm_rank
```

**特点**:
- 单调性严格满足 (α, β > 0 时)
- 可解释性强: "60% 来自教育阶段, 40% 来自词频难度"
- 同阶段内的区分度仅由 β × norm_rank 提供

#### B. 乘积公式

```
difficulty = norm_stage^α × norm_rank^β
```

**问题**:
- norm_stage = 0 (primary_3) 时 difficulty 恒为 0，不论 rank 多高
- 乘积效应会导致双中等阶段+中等rank的词被低估
- ❌ **不推荐**

#### C. 调和平均

```
difficulty = 2 × (α×norm_stage) × (β×norm_rank) / (α×norm_stage + β×norm_rank)
```

**问题**:
- 一方为 0 时结果恒为 0
- 非线性效应导致分数区间压缩
- ❌ **不推荐**

### 4.2 权重参数分析

| α | β | 跨阶段主导性 | 同阶段区分度 | 综合评价 |
|:-:|:-:|:-----------:|:-----------:|:--------:|
| 0.50 | 0.50 | 弱 (rank 与 stage 各半) | 强 (~0.47) | 同阶段区分好，但跨阶段不够清晰 |
| **0.55** | **0.45** | **中等** | **强 (~0.42)** | **推荐备选** |
| **0.60** | **0.40** | **较强** | **中等偏强 (~0.37)** | **⭐ 推荐 (首选)** |
| 0.65 | 0.35 | 强 | 中等 (~0.33) | 同阶段略微偏弱 |
| 0.70 | 0.30 | 很强 | 偏弱 (~0.28) | 同阶段区分度不足 |

#### 推荐: α = 0.60, β = 0.40

特点:
- **跨阶段**: norm_stage 占比 60%，确保不同教育阶段的主序关系
- **同阶段**: norm_rank 占比 40%，任何阶段内的最大 spread = 0.4 × (max_norm - min_norm) ≈ 0.23~0.37
- **平衡性**: 任何轴上都不会被对手完全淹没

### 4.3 预期分数分布 (α=0.6, β=0.4)

| 场景 | norm_stage | norm_rank | difficulty |
|------|:----------:|:---------:|:----------:|
| 最易 (primary_3, rank=1) | 0.000 | 0.067 | **0.027** |
| primary_3 中位数 | 0.000 | 0.708 | **0.283** |
| primary_3 最难 (rank=27103) | 0.000 | 0.990 | **0.396** |
| junior_7 中位数 | 0.400 | 0.662 | **0.505** |
| senior 中位数 | 0.700 | 0.795 | **0.738** |
| cet4 中位数 | 0.800 | 0.812 | **0.805** |
| cet6 中位数 | 0.900 | 0.856 | **0.882** |
| ielts 中位数 | 1.000 | 0.862 | **0.945** |

**观察**:
- primary 阶段: 0.027~0.396 (范围 0.37) ✅ 区分度良好
- senior 阶段: ~0.738 (中位), 但 rank=1 时 0.447, rank=30000 时 0.820 (范围 0.37)
- ielts 阶段: rank=1 时 0.627, rank=30000 时 1.000 (范围 0.37)
- 各阶段均在 ~0.37 范围内有区分

→ **根据 rank 反推估计优先级**, 基于实际数据的 first_stage priority vs rank 的分布:

使用分段平均 priority + 平滑插值:

```python

    (0, 4.5),          # rank=0 → priority ~4-5 (junior_7)
```

这是平滑插值版本，确保优先级随 rank 单调递增。

## 7. 备选方案

若未来需要调整:

- **提高区分度**: α=0.55, β=0.45 (同阶段 spread ~0.42)
- **加强阶段主导**: α=0.65, β=0.35 (同阶段 spread ~0.33)
- **改用 min-max 线性 rank**: 对 rank 分布做 rank/30000 线性归一化 → 但高频端几乎无区分度, 不推荐
- **加入置信度加权**: 未来可以根据 source_confidence 调整 (当用户需要时)
